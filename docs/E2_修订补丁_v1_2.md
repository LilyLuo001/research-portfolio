## 修订 1|论文主线重构:测度 + 压力测试为脊柱,H1 降级

v1.1 的隐含结构以 H1(循环杠杆对套息利差的敏感性)为核心因果主张。问题在 T8a 自己已点名:borrow_apy 由利用率经 IRM 机械内生决定,spread 对杠杆的"敏感性"在市场层面没有干净的识别来源——这正是 T13 审稿模拟中"描述性包装成因果"的教科书案例,任何计量补丁都只是缓解而非解决。

**新主线**(约束 T8 与 T12):

1. 论文的第一贡献 = **测度**(F1–F5 + 循环杠杆分解 λ + 持有人集中度),第二贡献 = **压力测试引擎与反事实**(T9),两者不依赖因果识别,是本文护城河。
2. H1 保留但降级为"敏感性/相关性证据",全部表述用 association 语言。
3. **唯一的因果主张**收窄为:货币政策高频冲击(Bauer–Swanson 类)× 敞口的 local projection——冲击外生性有文献背书,是全文识别上唯一站得住的部分。它从 T8a 的第 3 条子项**升格为 §5 的头条设定**。

**T8a prompt 修改**:第 1 条开头插入一句——"注意:本设计的输出将按'LP-on-monetary-shocks 为主设定、市场层面敏感性为辅助证据'的层级组织;主设定的识别论证按因果标准写,辅助设定禁止使用因果语言。" 第 4 条(IRM 内生性)的要求从"给出处理方案"改为"给出偏误方向分析 + 表述降级建议"。

**T12 prompt 修改**:章节结构调整为 概念框架 → Five Facts → 压力测试 → LP 因果检验 → 敏感性附录;C0 追加一条硬性规则:"辅助设定的正文表述白名单:is associated with / co-moves / consistent with;黑名单:causes / leads to / the effect of。"

**T13 prompt 修改**:附件 B 攻防表中新增一行,把"H1 内生性"标记为【已通过降级处理】,让审稿模拟去攻击降级后的表述是否仍然过强。

## 修订 2|新增 T15:快速描述性短文(占坑)

**动因**:本领域以月为单位迭代,v1.1 把全部产出压在流水线末端,撞车风险(kill criterion #3)不对称地大。F1–F5 + 集中度事实在 panel 建成后即可成文,一篇 FEDS-Note/BIS-Bulletin 体量(8–12 页)的描述性短文能:①在 SSRN 占坑锁定优先权;②本身就是接触发行方与协议方(行业合作)的最强敲门砖;③给 T11 扫描到的潜在竞争者一个"此坑有主"的信号。

**排期**:panel_daily 通过 15 条断言后立即启动,目标全程 ≤ 3 周。

**指派**:正文 → claude.ai 会话(Pro 额度);图表复用 T7 脚本;数字注入与 T12 相同的占位符机制;红队一轮即可(DeepSeek 推理档)。

```
【粘贴 C0】
任务(T15): 撰写 8–12 页英文描述性短文(FEDS Note / BIS Bulletin 体例),
题材 = F1–F5 + 持有人集中度 + λ 分解的核心事实, 不含 H1、不含压力测试、
不含任何因果表述。
输入: facts_narrative.md、results/facts.json 键名清单(无数值)、
T1 制度事实表、references.bib。
硬性规则: 与 T12 完全相同的三条(占位符数字、bib 白名单、主张挂钩);
另加: 通篇禁用因果与预测语言, 结论段只允许"提出问题"不允许"回答问题"
(把 H1 与压测明确标注为 ongoing work, 起到占坑而非泄底的作用)。
交付: 全文 + 占位符清单 + 一页的发行方/协议方分发名单建议
(按 F1–F5 中每个事实最相关的机构列出)。
```

## 修订 3|T9a 压测引擎产品化条款

压测引擎是本项目商业价值最高的产物(Gauntlet/Chaos 类风控公司的核心业务即此)。T9a 的设计已经 90% 满足产品化(registry.yaml / scenarios.yaml 外置参数化),补齐最后 10%:

**T9a prompt 追加**:

```
7. 产品化约束: stress/ 模块对外只暴露三个稳定接口(load_positions,
   run_scenario, report), 接口签名写入 manifest 并视为契约(变更需人工
   签字); 引擎核心不得 import 本 repo 的论文分析代码(单向依赖);
   附 INTERFACE.md 说明第三方(协议方/风控方)如何用自己的 registry 与
   scenarios 运行引擎。
```

## 修订 4|推断预承诺:随机化推断为主

v1.1 中 wild cluster bootstrap 与随机化推断并列为备选。有效聚类数 < 10 时 wild bootstrap 自身也不稳。**预承诺(写入 T8a 设计文档的第 2 条)**:主推断 = 随机化推断(置换单元与次数由设计文档定),wild bootstrap 与解析双向聚类降为参考列。在数据打开前锁定,避免事后择优。
