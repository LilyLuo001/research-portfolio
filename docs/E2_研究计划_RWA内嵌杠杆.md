# 《链上再抵押：代币化真实世界资产中的内嵌杠杆——测量、机制与脆弱性》

*Onchain Rehypothecation: Embedded Leverage in Tokenized Real-World Assets — Measurement, Mechanisms, and Fragility*

---

## 0. 一页纸摘要

**研究问题**：代币化真实世界资产（RWA）进入 DeFi 借贷协议后，被大规模用于"存抵押→借稳定币→买更多 RWA→再存抵押"的循环杠杆策略。本文对这一现象做**首次系统性测量**，回答四个问题：(1) RWA 的链上"采用量"中有多大比例是循环杠杆的机械放大？(2) 内嵌杠杆的水平、分布与资金来源是什么——谁在为机构的杠杆循环提供融资？(3) 杠杆对利差（RWA 收益率 − 稳定币借款利率）的敏感性如何，货币政策如何通过这条新渠道传导？(4) 在"抵押品无二级市场、清算机制事实上失灵"的制度下，压力情景会造成多大损失、由谁承担？

**核心概念贡献**："**没有流动性后盾的流动性转换**"（liquidity transformation without a liquidity backstop）。DeFi 借贷的清算机制假设清算人能在公开市场卖出抵押品；但许可型 RWA 没有二级市场，清算人必须是白名单地址、退出只能走发行方 T+N 申赎且可能有赎回门槛。于是标准的超额抵押安全叙事在 RWA 上是断裂的：贷方（其中包括经 Coinbase 等渠道进入 Morpho 金库的零售 USDC）实际持有的是对"申赎通道畅通"这一假设的敞口。这是本文区别于"又一篇 DeFi 描述"的关键。

**论文类型定位**：测量型 + 机制型（"把民间传说变成精确事实"的路线——领域内人人知道 looping 存在，无人量化过其规模、融资结构与脆弱性）。不以因果识别为主卖点，但内置一个准实验时刻（货币政策冲击 → 利差 → 杠杆）。

**目标发表**：上限 RFS / JFE（若测出的事实足够 first-order）；现实靶区 JFQA、Journal of Financial Intermediation、Management Science；政策副产品投 BIS Working Paper / FSB 讨论渠道。

## 1. 研究问题与贡献定位

### 1.1 三层研究问题

**Q1（测量）**：定义并估计——
- 策略层杠杆乘数 λ_v(t) = 金库/策略 v 的链上 RWA 抵押品总额（gross）÷ 外部净本金（net external principal）；
- 系统层内嵌杠杆 Λ(t) = Σ gross ÷ Σ net，及其跨资产、跨协议、跨链分布；
- "采用量虚增率" I(t) = 1 − net/gross：RWA 头条 TVL 中由循环放大的比例。

**Q2（融资结构与期限错配）**：借贷市场贷方是谁（零售渠道金库 vs 机构地址 vs 协议自有），贷方可即时提取的 USDC 与抵押品 T+N 申赎之间的期限错配有多大，Public Allocator 等跨市场流动性调拨在压力日的实际作用。

**Q3（杠杆的决定因素）**：λ 对利差 s = y_RWA − r_borrow 的弹性；FOMC 高频货币政策意外作为 r_borrow 的外生推动，检验"链上套息渠道"的货币政策传导。

**Q4（脆弱性）**：给定真实 LLTV、oracle 机制与零二级流动性，反事实压力情景（NAV 下修、利率冲击、赎回门槛触发）下的清算级联/坏账社会化损失估计；oracle 更新的离散性（NAV 仅工作日更新 vs 借贷 24/7 计息）造成的可预测清算时点与陈旧价格借款能力。

### 1.2 对文献的贡献

1. **影子银行/再抵押文献**（Singh & Aitken 的再抵押测量；Gorton–Metrick 证券化银行；Krishnamurthy 等回购挤兑）：链上数据首次允许**逐笔、逐地址、无抽样**地观察再抵押链条——传统再抵押研究最大的痛点（数据不可见）在这里不存在。这是本文的"不公平优势"。
2. **DeFi 借贷文献**（Lehar–Parlour；OECD/学术界的清算研究；Aave/Compound 实证）：现有清算文献全部以"抵押品可在 DEX 即时变现"为前提；本文指出并量化这一前提在 RWA 上失效后的风险再分配。
3. **货币政策传导新渠道**：链上套息利差是政策利率的直接函数，提供一个高频、可观察的"利率 → 杠杆"传导实验室。
4. **政策**：GENIUS/CLARITY、MiCA 语境下监管者最关心的正是"代币化资产 + DeFi 组合性 = 隐性杠杆积累"，本文提供第一份体检报告。

### 1.3 为什么是现在、为什么是我们

- 事件与规模刚刚跨过可研究门槛：Morpho RWA 市场 2025 年从 0 增至约 2.8 亿美元（mF-ONE 约 1.9 亿居首），Aave Horizon 2025-08 上线，sACRED/FalconX CV/mF-ONE 等循环策略均已运行 6–18 个月，有足够时间序列。
- 无人占坑（截至本计划撰写时，SSRN/arXiv 无系统测量 RWA 循环杠杆的论文；需每两周复查一次，见 §9 kill criteria）。
- 全部核心数据免费公开在链上，不依赖任何机构数据授权。

## 2. 制度背景与概念框架

### 2.1 循环策略的机械结构

以 sACRED/Morpho 为代表：用户（经 KYC 白名单）存入 RWA 代币 → 金库以其为抵押借出 USDC（目标 LTV = u）→ 用 USDC 申购更多 RWA → 再存入。极限杠杆：

> λ = 1 / (1 − u)

u = 0.77 时 λ ≈ 4.3；u = 0.86 时 λ ≈ 7.1。实际 λ 低于极限值（风控留 buffer），由 curator（Gauntlet、Steakhouse 等）的优化引擎动态设定。

### 2.2 清算距离与"断裂的清算"

设清算阈值 LLTV = ℓ，当前 LTV = u，NAV 下跌比例 d 使 LTV 变为 u/(1−d)。触发清算的临界跌幅：

> d* = 1 − u/ℓ

u = 0.80、ℓ = 0.86 时 d* ≈ 7%。对 T-bill 基金这是久期上几乎不可能的单日跌幅；但对 CLO（JAAA）与私募信贷（ACRED、mF-ONE、FalconX CV），NAV 下修 5–10% 是信用周期内的现实事件。**关键制度事实**：即便触发，清算人（a）须在发行方白名单内，（b）无法在二级市场卖出，只能走发行方申赎（T+1 至 T+N，开放式基金可能设 gate）。清算激励结构假设的"即时折价变现"不存在 → 可能出现无人清算 → 坏账按 Morpho 机制社会化给该市场全体贷方。

### 2.3 期限/流动性错配的三层结构

| 层 | 承诺 | 实际流动性 |
|---|---|---|
| 贷方（USDC 存款人，含经零售渠道进入的资金） | 随时提取 | 受市场利用率约束；利用率→100% 时排队 |
| 借款人（循环策略） | 随时去杠杆 | 去杠杆需赎回 RWA，受 T+N 与 gate 约束 |
| 抵押品（RWA 基金份额） | oracle 按 NAV 计价 | 无二级市场；NAV 为申报值，非成交价 |

这就是经典的银行式流动性转换，但没有存款保险、没有最后贷款人、清算机制名存实亡——本文的概念主线。

### 2.4 Oracle 层

Aave Horizon 采用 Chainlink SmartData/NAVLink 喂 NAV；Morpho 各市场 oracle 由创建者指定（可能是 NAVLink、可能是发行方直喂、可能是硬编码）。NAV 通常每个工作日更新一次，而借款利息 24/7 复利 → 周末/节假日 LTV 只因利息单调恶化、抵押价值冻结；周一 NAV 跳更新造成离散的清算风险时点。这一"陈旧计价 + 连续计息"结构本身是可检验的机制（H4）。

### 3.3 数据工程的两个关键难点（提前声明）

1. **闪电贷循环 vs 迭代循环**：单交易内完成的循环只有 call trace 能拆开（Transfer 事件序列不足以判定资金因果）；跨交易迭代循环用"同地址 k 区块窗口内 borrow→申购→supply"启发式识别。两者给出杠杆分解的上下界（§5.1）。
2. **跨链去重**：Wormhole 受控 burn/mint 会造成 gross 重复计数；以发行方总供给为约束做跨链净额调和，与 RWA.xyz、DeFiLlama 三方核对。

## 4. 测量方法：外部本金与循环放大的分解

### 4.1 分解算法（论文技术核心）

对每个策略金库 v、每个抵押品供给事件 e：

1. **同交易溯源**：若供给的 RWA 在同一交易内由"borrow USDC → 发行方申购/一级认购合约 mint"资金链产生 → 标记为循环供给（闪电型）。
2. **短窗溯源**：若供给地址在前 k 个区块内（k ∈ {同交易, 50, 250, 1800≈6h}）有同市场 borrow 且金额匹配（±5% 容差，扣除 gas 与费率）→ 循环供给（迭代型）。
3. **其余** → 外部本金供给。
4. 汇总得 gross_v(t)、net_v(t)、λ_v(t) = gross/net；对每个 k 报告，主结果用 k=250，其他为稳健性。
5. **验证**：对 Gauntlet/Steakhouse 公开披露过目标杠杆的策略（如 sACRED 策略文档），比对算法估计 λ 与披露目标——这是算法的外部效度检验，也是与业界建立联系的天然接口。

### 4.2 输出的核心测量对象

- λ_v(t) 面板（策略×日）；Λ(t) 系统序列；I(t) 虚增率；
- 再抵押链长分布（RWA 被跨协议二次抵押的情形，如同一底层经包装后出现在两个协议）；
- 贷方构成矩阵：市场×贷方类型（零售渠道金库 / 机构 / 其他金库）×日；
- 错配指标 M_v(t) = 可即时提取贷方资金 ÷ 抵押品 T+N 可变现额；
- oracle 陈旧度 τ_stale(t) = 当前时刻距最近 NAV 更新的小时数。

## 5. 实证设计

### 5.1 第一部分：五个典型事实（Facts 1–5）

- **F1 规模与虚增**：RWA 链上借贷抵押总额的 gross/net 分解；估计头条"RWA 采用量"中 X% 为循环放大。预期这是论文的头条数字。
- **F2 杠杆分布**：λ 的横截面（按资产类别：T-bill vs CLO vs 私募信贷）与时间演化；与 LLTV 极限值的距离（风控 buffer）。
- **F3 谁在融资**：贷方分解——特别是经零售渠道（如 Coinbase USDC 出借经由 curator 金库）流入 RWA 循环市场的比例。若显著，"零售活期资金融资机构私募信贷杠杆"本身就是政策级事实。
- **F4 期限错配**：M_v(t) 的水平与压力日行为；利用率触顶（→100%）事件清单与提取排队时长。
- **F5 Oracle 离散性**：NAV 更新频率、周末冻结、周一跳变分布；陈旧计价窗口内的新增借款量（"以旧价借新钱"）。

### 5.2 第二部分：杠杆的决定因素（H1）

**H1**：循环杠杆随套息利差扩大而上升、随利差压缩而去化。

基准回归（策略×日面板）：

> Δlog λ_{v,t} = β₁ s_{v,t−1} + β₂ Δr^{borrow}_{v,t} + β₃ Δy^{RWA}_{v,t} + γ′X_{v,t} + α_v + δ_t + ε_{v,t}

X 含利用率、金库容量上限变更（curator 参数事件，从治理/链上参数读取）、TVL。标准误按 curator 与资产双向聚类（有效聚类少 → 同时报告 wild cluster bootstrap）。

**识别加强**：r_borrow 的政策成分用 FOMC 高频意外 mps_t 做冲击：

> Δlog λ_{v,t∈[0,h]} = β^h · mps_t × exposure_v + …（local projection，h = 1…30 天）

exposure_v = 策略对浮动借款利率的事前敏感度。这给出"货币政策 → 链上套息杠杆"的传导弹性，是论文里最接近因果的一块。

**竞争性解释与排除**：(a) 激励代币空投驱动而非利差驱动——收集 Morpho/协议激励计划日历作控制；(b) curator 主动调参而非市场内生——参数变更事件单独建模并做剔除样本稳健性。

### 5.3 第三部分：脆弱性与反事实压力测试（H3/H4）

**压力情景引擎**（基于真实链上参数逐头寸模拟）：

1. **情景 A（信用冲击）**：JAAA/私募信贷 NAV 下修 {3%, 5%, 10%}（校准自 2020-03 CLO AAA 与 BDC 折价历史）；
2. **情景 B（利率冲击）**：借款利率 +{200, 500}bp（校准自链上利用率-利率曲线在挤兑时的实际形态）；
3. **情景 C（申赎冻结）**：发行方触发 gate，T+1 变 T+30；
4. **情景 D（复合）**：A+C 同时发生（信用事件通常伴随赎回限制——开放式基金文献的标准事实）。

每情景输出：触发清算的头寸比例、理论清算量 vs 白名单清算人容量、无人清算下的坏账额、坏账社会化后贷方损失率（分贷方类型——零售渠道资金的损失敞口单列）、跨市场传染（同 curator 金库的共同贷方通道）。

**H4（oracle 机制）**：清算与濒临清算事件在 NAV 更新时点的聚集性（围绕更新时间戳的事件研究）；周末新增借款是否系统性利用陈旧计价（比较周五更新后 vs 周一更新前的借款 LTV 分布）。

### 5.4 第四部分：与加密原生抵押品的对照

同一套测量施加于 wstETH/cbBTC 循环：RWA 循环的独特性在于（预期）更高的 λ（因低波动 LLTV 更高）、零二级流动性、更强的期限错配——用三组对照回归把"RWA 特殊性"钉死，防审稿人一句"looping 在 DeFi 里司空见惯"。

## 6. 稳健性清单

**测量稳健性**
1. 分解窗口 k ∈ {同交易, 50, 250, 1800 区块}，报告 λ 的上下界带；
2. 金额匹配容差 {±2%, ±5%, ±10%}；
3. 地址聚类：无聚类 / 启发式共控聚类 / 商业标签（如可得），三版结果；
4. 数据源三角验证：自建索引 vs Dune vs subgraph 的总量对账（差异 >2% 的市场逐一排查）；
5. 跨链去重前后对照；包装代币 1:1 映射的链上验证（mint/burn 守恒检查）；
6. 含死亡/清零市场的全样本 vs 过滤样本。

**回归稳健性**
7. 双向聚类 + wild cluster bootstrap + 随机化推断（curator 数少）；
8. 水平 vs 对数 vs 一阶差分设定；winsorize 1%/5%；
9. 剔除单一最大策略（mF-ONE）后的全部主结果——右偏分布下这是审稿人必问；
10. 激励计划日历控制的进出；curator 参数变更事件的剔除样本；
11. local projection 的滞后阶与窗口敏感性；安慰剂冲击日。

**模拟稳健性**
12. NAV 冲击幅度网格；清算人容量假设 {0, 已观察白名单容量, 无限}；利率曲线形态的历史极值替代；
13. 与真实压力日（样本内利用率触顶事件）的"事后预测"比对——模拟引擎若不能复现已发生的小型挤兑，参数需重校准。
